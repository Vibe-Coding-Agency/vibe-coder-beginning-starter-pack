# 📚 Glossary

Quick definitions for terms you will see throughout the starter pack.

- **Agent** — A system that uses a model in a loop to decide and execute actions.
- **Chunking** — Splitting documents into smaller pieces for embedding and retrieval.
- **Context window** — The maximum number of tokens a model can process in one call.
- **Embedding** — A vector that represents the meaning of text or other data.
- **Evaluation** — A method for measuring how well your system performs.
- **Guardrail** — A rule or check that validates inputs or outputs.
- **Hallucination** — When a model generates false or unsupported information.
- **Inference** — Running a model to get a prediction.
- **LLM** — Large language model.
- **Metric** — A number used to compare performance.
- **Pipeline** — A fixed sequence of steps for processing data.
- **Prompt** — The input text given to a language model.
- **Prompt injection** — An attack that tries to override instructions in the prompt.
- **RAG** — Retrieval-augmented generation; fetching relevant context before generating.
- **Re-ranking** — Scoring retrieved chunks to find the most relevant ones.
- **Regression** — When a change makes performance worse on previously good cases.
- **Token** — A unit of text used by language models.
- **Trace** — A record of everything that happened during one request.
- **Vector store** — A database optimized for storing and searching embeddings.
